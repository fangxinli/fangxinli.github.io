使用 Hexo 构建个人博客站点

  • 学习 前端 hexo
  • 2018-05-30

安装构建工具

全局安装 hexo-cli

检验版本信息

version

显示版本信息。

初始化站点

init

初始化站点。如果没有提供 folder，Hexo会在当前目录下创建站点。

新建文章页面

new

  • layout：可选的，指定创建文件的类型，类型包括：post（文章）、page（页面）、draft（草稿）。如果没有指定该参数，则使用 _config.yml 配置文件 default 指定的类型。
  • title：必须的，指定文章的标题，如果标题中包含空格，则需要使用引号包裹标题。

在新建文章时，hexo会去脚手架目录（scaffolds）查找是否有指定类型的脚手架文件，如果有则使用该文件作为模板创建新的文件。

发布文章草稿

publish

发布草稿。

该命令会根据 layout 参数的不同，将 _draft 文件夹中的草稿发不成文章（移动到 _posts 文件夹中），或作为一个新建的页面生成。

生成静态文件

generate

生成静态文件。

选项 描述
-d，--deploy 生成静态文件之后部署
-w，--watch 监视文件的改变
-b，--bail Raise an error if any unhandled exception is thrown during generation
-f， --force 强制生成

清除静态文件

clean

清除编译之后的结果，删除public目录

查看站点信息

list

列出指定类型的信息，可用的类型有：page，post，route，tag，category。

本地预览站点

server

开启一个本地服务器。默认情况下，是在：http://localhost:4000/。

  2. 将下面的文本粘贴到 Git Bash 中，然后把邮箱修改成你的 Github Email 地址。

  3. 确认 key 放到哪，直接按回车（Enter），表示接受默认位置。

  4. 设置安全口令

添加认证密钥

将生成的 SSH key 添加到你的 Github 账户

  1. 将 SSH key 复制到粘贴板。

  2. 打开 Github 的设置页面。

  3. 点击侧边栏中的 SSH and GPG keys。

  4. 点击 New SSH key。

  5. 将 SSH key 粘贴到输入框中。

  6. 点击 Add SSH key。

  7. 输入 Github 账户密码确认前面的操作。

部署站点

deploy

部署你的站点。
